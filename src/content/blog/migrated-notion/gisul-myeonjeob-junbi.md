# 예상 질문 리스트업


### **퍼블리셔에서 프론트엔드로 전향하신 이유가 궁금해요.**

- 단순히 정적인 UI를 넘어서, **데이터를 다루며 사용자 경험을 주도적으로 만들고 싶어서** 프론트엔드로 전향하게 됐습니다.

**1. 브라우저의 렌더링 과정은 어떻게 되나요?**

- 브라우저는 HTML과 CSS를 파싱하여 DOM과 CSSOM을 생성하고, 이를 결합해 렌더 트리를 만든 후, 레이아웃과 페인팅을 거쳐 화면에 표시합니다.

**2. HTTP와 HTTPS의 차이점은 무엇인가요?**

- HTTP는 데이터를 평문으로 전송하지만, HTTPS는 SSL/TLS 프로토콜을 통해 데이터를 암호화하여 전송하므로 보안성이 높습니다.

**2. URL을 입력하면 어떤 일이 발생하나요?**

- 사용자가 URL을 입력하면 DNS 조회를 통해 IP 주소를 찾고, 서버와 TCP 연결을 설정한 후, HTTP 요청과 응답을 주고받아 브라우저가 콘텐츠를 렌더링합니다.

**3. CORS란 무엇이며, 어떻게 대응하나요?**

- CORS는 다른 출처의 리소스 접근을 제어하는 보안 기능으로, 서버에서 적절한 헤더를 설정하여 브라우저의 요청을 허용할 수 있습니다.

**4. RESTful API란 무엇인가요?**

- RESTful API는 HTTP 메서드와 URL을 활용하여 리소스를 정의하고 조작하는 아키텍처 스타일입니다.

**5. GET과 POST의 차이점은 무엇인가요?**

- GET은 데이터를 요청할 때, POST는 데이터를 서버에 제출할 때 사용하며, GET은 URL에 데이터를 포함하고 POST는 본문에 데이터를 포함합니다.

**6. PUT과 PATCH의 차이점은 무엇인가요?**

- PUT은 리소스의 전체를 업데이트하고, PATCH는 리소스의 일부만을 업데이트합니다.

**7. 크로스 브라우징이란 무엇인가요?**

- 크로스 브라우징은 다양한 브라우저에서 웹 페이지가 일관되게 동작하도록 개발하는 것을 의미합니다.

**8. CSR과 SSR의 차이점은 무엇인가요?**

- CSR은 클라이언트 측에서 렌더링을 수행하고, SSR은 서버 측에서 렌더링을 수행하여 클라이언트에 완성된 HTML을 전달합니다.

**9. DOM과 가상 DOM이란 무엇인가요?**

- DOM은 문서의 구조화된 표현을 제공하는 프로그래밍 인터페이스이고, 가상 DOM은 실제 DOM의 변경 사항을 최소화하기 위해 메모리에 존재하는 가벼운 사본입니다.

**10. ‘attribute’와 ‘property’의 차이점은 무엇인가요?**

- ‘attribute’는 HTML 문서의 정적인 속성이며, ‘property’는 DOM 트리에서 동적으로 변경 가능한 속성입니다.

**11. Flexbox와 Grid의 차이점은 무엇인가요?**

- Flexbox는 1차원 레이아웃 시스템으로 요소들을 주축과 교차축 방향으로 배치하고, Grid는 2차원 레이아웃 시스템으로 행과 열을 사용하여 복잡한 레이아웃을 구성합니다.

**12. JavaScript의 클로저(closure)란 무엇인가요?**

- 클로저는 함수와 함수가 선언된 렉시컬 환경의 조합으로, 함수가 외부 함수의 범위 내에 있는 변수에 접근할 수 있게 합니다.

**13. ‘this’ 키워드의 의미는 무엇인가요?**

- ‘this’는 현재 실행 중인 컨텍스트를 가리키며, 함수 내에서는 호출한 객체를, 전역 컨텍스트에서는 전역 객체를 참조합니다.

**14. 비동기 프로그래밍과 콜백, 프로미스, async/await의 차이점은 무엇인가요?**

- 비동기 프로그래밍은 코드가 블로킹되지 않고 다음 작업을 진행하는 방식이며, 콜백은 함수에 다른 함수를 인수로 전달하는 것이고, 프로미스는 비동기 작업의 완료를 나타내는 객체이며, async/await는 프로미스를 간결하게 다루는 구문입니다.

**15. 이벤트 버블링과 이벤트 캡처링의 차이점은 무엇인가요?**

- 이벤트 버블링은 이벤트가 가장 깊은 요소에서 시작하여 부모 요소로 전파되는 것이고, 이벤트 캡처링은 이벤트가 상위 요소에서 시작하여 목표 요소로 전파되는 것입니다.

**16. React의 상태(state)와 속성(props)의 차이점은 무엇인가요?**

- 상태(state)는 컴포넌트 내에서 관리되는 동적인 데이터이고, 속성(props)은 부모 컴포넌트가 자식 컴포넌트에 전달하는 변경 불가능한 데이터입니다.

**17. SPA와 MPA의 차이점은 무엇인가요?**

- SPA는 하나의 HTML 페이지로 구성되어 페이지 간 이동 시 전체 페이지를 새로 로드하지 않고, MPA는 각 페이지가 별도의 HTML 문서로 구성되어 페이지 간 이동 시 전체 페이지를 새로 로드합니다.

**18. CSS 전처리기(SASS, LESS)란 무엇인가요?**

- CSS 전처리기는 CSS를 더 효율적으로 작성할 수 있게 도와주는 도구로, SASS와 LESS는 변수, 중첩, 믹스인 등 다양한 기능을 제공하여 코드의 재사용성과 유지보수성을 높입니다.

**19. 웹 접근성이란 무엇이며, 왜 중요한가요?**

- 웹 접근성은 장애가 있는 사람들이 웹 콘텐츠를 접근하고 사용할 수 있도록 하는 것으로, 모든 사용자에게 공평한 접근을 보장하고 더 넓은 사용자 기반을 확보하는 데 중요합니다.

**20. AJAX란 무엇인가요?**

- AJAX는 웹 페이지를 전체적으로 새로 고치지 않고도 서버와 데이터를 주고받을 수 있게 해주는 기술로, 더 빠르고 동적인 사용자 경험을 제공합니다.

**쿠키와 세션의 차이점은 무엇인가요?**


쿠키는 클라이언트 측에 저장되는 데이터로, 사용자의 브라우저에 저장되어 매 요청 시 서버에 전송됩니다. 세션은 서버 측에서 관리되는 데이터로, 클라이언트와의 세션 ID를 통해 식별됩니다.


**자바스크립트의 이벤트 루프란 무엇인가요?**


이벤트 루프는 자바스크립트의 비동기 처리를 가능하게 하는 구조로, 콜 스택과 태스크 큐를 모니터링하며, 콜 스택이 비어 있을 때 태스크 큐에서 대기 중인 콜백을 실행합니다.


**프로토타입과 프로토타입 체인이란 무엇인가요?**


프로토타입은 자바스크립트 객체의 상위 객체를 가리키며, 프로토타입 체인은 객체가 특정 프로퍼티나 메서드를 찾을 때 상위 프로토타입을 따라가며 검색하는 구조입니다.


**ES6의 화살표 함수와 일반 함수의 차이점은 무엇인가요?**


화살표 함수는 ‘this’ 바인딩이 없으며, 상위 스코프의 ‘this’를 그대로 사용합니다. 반면, 일반 함수는 호출 방식에 따라 ‘this’가 동적으로 결정됩니다.


**호이스팅이란 무엇인가요?**


호이스팅은 자바스크립트에서 변수와 함수 선언이 해당 스코프의 최상단으로 끌어올려지는 현상을 말합니다.


**이벤트 위임이란 무엇인가요?**


이벤트 위임은 하위 요소의 이벤트를 상위 요소에서 처리하는 기법으로, 이벤트 리스너를 상위 요소에 등록하여 하위 요소의 이벤트를 관리합니다.


**웹팩과 바벨의 역할은 무엇인가요?**


웹팩은 모듈 번들러로, 여러 개의 자바스크립트 파일을 하나의 파일로 묶어줍니다. 바벨은 자바스크립트 컴파일러로, 최신 문법을 구 버전의 브라우저에서도 동작하도록 변환합니다.


**PWA(Progressive Web App)란 무엇인가요?**


PWA는 웹 애플리케이션에 네이티브 앱과 유사한 사용자 경험을 제공하기 위한 기술로, 오프라인 지원, 푸시 알림, 홈 화면 추가 등의 기능을 포함합니다.


**웹 소켓(WebSocket)이란 무엇인가요?**


웹 소켓은 클라이언트와 서버 간의 전이중 통신을 가능하게 하는 프로토콜로, 실시간 데이터 전송에 사용됩니다.


**서비스 워커(Service Worker)의 역할은 무엇인가요?**


서비스 워커는 브라우저와 웹 서버 사이에서 프록시 역할을 수행하는 스크립트로, 오프라인 지원, 캐싱, 백그라운드 동기화 등의 기능을 제공합니다.


**웹 컴포넌트(Web Components)란 무엇인가요?**


웹 컴포넌트는 재사용 가능한 커스텀 요소를 만들기 위한 기술로, Shadow DOM, HTML 템플릿, 커스텀 엘리먼트 등의 기술을 포함합니다.


**상태 관리 라이브러리(Redux, MobX 등)를 사용하는 이유는 무엇인가요?**


상태 관리 라이브러리는 애플리케이션의 상태를 일관되게 관리하고, 컴포넌트 간의 상태 공유를 용이하게 하여 복잡한 상태 관리를 단순화합니다.


**웹 성능 최적화를 위한 방법에는 어떤 것들이 있나요?**


코드 스플리팅, 이미지 최적화, 지연 로딩, 캐싱, CDN 사용 등 다양한 기법이 있습니다.


**SEO(검색 엔진 최적화)를 위해 고려해야 할 사항은 무엇인가요?**


메타 태그 최적화, 시맨틱 마크업 사용, 모바일 친화성, 로딩 속도 개선 등 다양한 요소를 고려해야 합니다.


**웹 보안에서 XSS와 CSRF는 무엇인가요?**


XSS는 악성 스크립트를 삽입하여 사용자 정보를 탈취하는 공격이며, CSRF는 사용자가 의도하지 않은 요청을 보내도록 유도하는 공격입니다.
