# 변수란?

## 변수

- 값을 저장하기위해 확보한 메모리 공간 자체
- 메모리 공간을 식별하기 위해 붙인 이름
- 값의 위치를 가리키는 상징적인 이름

## 식별자

- 어떤 값을 구별해서 식별할 수 있는 고유한 이름
- 식별자는 값이 아니라 메모리 주소를 기억하고 있다.
- 변수,함수,클래스도 식별자

## 변수 선언

- 변수 선언은 변수를 생성하는 것
- 값을 저장하기 위한 메모리 공간을 확보하고 확보된 메모리 공간의 주소를 연결해서 값을 저장할 수 있게 준비된 상태
- 변수 선언 키워드는 var, let, const 키워드를 사용한다.

### 변수 선언문

```jsx
var score; // 변수 선언
```

### 변수 선언의 수행 단계

1. 선언 단계 : js 엔진에 변수의 존재를 알린다
2. 초기화 단계 : 저장하기 위한 메모리 공간을 확보하고 암묵적으로 undefined를 할당해 초기화함.
- 예외적인 var의 경우는 선언 단계와 초기화 단계가 동시 진행 된다

## 변수 선언의 실행 시점과 변수 호이스팅

### var의 호이스팅

```jsx
console.log(score); // undefined
var score; // 변수 선언문
```

- 위 코드에서는 실행 순서로 보았을땐 상단에서 순차적으로 진행이 되면서 console.log(score);는 score 변수의 선언이 실행되기 전이라 사실상 참조 에러가 발생할 것 같지만 undefined가 출력된다.
- 이러한 이유는 ‘**변수 선언이 런타임이 아니라 그 이전 단계에서 먼저 실행되기 때문**’ 이다.
- 이처럼 ‘**변수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트 고유의 특징**’을 ‘변수 호이스팅’ 이라 한다.

## 값의 할당

- 변수에 값을 할당 할 때에는 할당 연산자 = 를 사용한다.
1. 예제-1

```jsx
var score; // 변수 선언
score = 80; // 값의 할당
```

1. 예제-2 (’문’으로 단축 표현)

```jsx
var score = 80; // 변수 선언과 값의 할당
```

- 위의 단축 표현한 코드는 변수 선언과 할당을 나누어 표현한 코드와 동일하게 동작한다.
- 다만, 변수 선언과 값의 할당의 실행 시점은 다르기 때문에 ‘**변수 선언은 소스코드가 런타임 이전에 먼저 실행되지만(=호이스팅) 값의 할당은 소스코드가 런타임 시점에 실행된다**’.

```jsx
console.log(score); // 결과 : undefined

var score = 80;

console.log(score); // 결과 : 80
```

- 호이스팅 결과

```jsx
console.log(score); // 결과 : undefined

score = 80;
var score;

console.log(score); // 결과 : 80
```

### 값의 재할당

재할당은 변수에 저장된 값을 다른 값으로 변경한다.

```jsx
var score = 80;
score = 90;

console.log(score); // 90
```

- 상수(const 키워드)
- 상수란 ‘**값을 재할당할 수 없어서 변수에 저장된 값을 변경 할 수 없다면 변수가 아니라 상수 라고 한다**’
- 위 예제 중 var score로 값을 할당한 것이 아닌 const score로 할당했을 경우 재할당을 시도하면 타입에러가 발생한다.

# 식별자

## 네이밍 규칙

1. 특수문자를 제외한 문자. 숫자, 언더스코어(_),달러 기호($)를 포함할 수 있다.
2. 단 숫자로 시작하는 것은 허용하지 않는다.
3. 예약어는 식별자로 사용할 수 없다.

### 특징

- 자바스크립트는 대소문자를 구별한다

```jsx
var firstname;
var firstName;
var FIRSTNAME;

// 위 셋은 각각 별개의 변수이다.
```

### 네이밍 컨벤션

- 변수나 함수의 이름에는 카멜 케이스 사용

```jsx
var firstName;

function secoendFunction(){}
```

- 생성자 함수, 클래스 이름

```jsx
class first_name(){}
```
