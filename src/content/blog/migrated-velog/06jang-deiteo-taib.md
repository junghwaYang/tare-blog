# 데이터 타입의 종류

## 원시타입

### 숫자 타입

- 숫자타입의 경우 정ㅇ수만을 위한 타입이 없고 모든 수를 실수로 처리한다.

```jsx
console.log(1 === 1.0); // true
```

### 문자열 타입

문자열은 작은따옴표, 큰따옴표 또는 백틱으로 텍스트로 감싸서 표현한다.

가장 일반적인 표현법은 작은 따옴표를 사용한다.

- 백틱을 이용한 템플릿 리터럴도 있다.

### 불리언 타입

불리언 타입의 값은 논리적 참, 거짓을 나타내는 true와 false 뿐이다.

### undefined 타입

var 로 선언한 변수는 undefined로 초기화 된다.

undefined은 값이 없는것이 아닌 변수를 초기화 할 때 사용하는 값이다.

### null 타입

null은 변수에 값이 없다는 것을 의도적으로 명시 할 때 사용된다.

### 심벌 타입

심벌타입은 변경 불가능한 원시 타입의 값이다.

불변의 값 > db key의 의미를 갖고있다고 보면 된다.

중복되지않는 유일무이한 값

## 객체 타입

자바스크립트를 이루고 있는 거의 모든것이 객체이다.

# 데이터 타입의 필요성

- 값을 저장 할 때 확보해야하는 **메모리 공간의 크기를 결정**하기 위해
- 값을 참조할 때 한번에 읽어 들여야 할 **메모리 공간의 크기를 결정**하기 위해
- 메모리에서 읽어 들인 2진수를 어떻게 해석할지 결정하기 위해

## 동적 타이핑

### 정적 타입 언어

C와 같은 변수를 선언할때 변수에 할당할 수 있는 데이터 타입을 사전에 선언해야한다.

정적 타입 언어는 변수의 타입을 변경 할 수 없다.

### 동적 타입 언어

동적 타입언어는 정적 타입 언어와 다르게 재할당에 의해 변수의 타입은 언제든지 동적으로 변할 수 있다.

동적 타입 언어의 변수는 선언이 아닌 할당에 의해 타입이 결정 된다.

**동적 타입 언어는 유연성은 높지만 단점으로는 신뢰성이 떨어진다.**

이유는 재할당에 의해 변수의 타입은 언제든지 변할 수 있기때문에 개발의 오류를 줄이기 위해서는 명확한 식별자 네이밍을 쓰는 것을 요한다.
