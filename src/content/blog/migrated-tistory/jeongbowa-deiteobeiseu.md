## 2. 파일시스템

### 01      파일시스템

> 데이터를 파일에 저장 후 수정,삭제,검색하는 기능을 제공하여 사용자가 쉽게 사용하게끔 지원하는 프로그램   

#### 파일시스템 특징

- 응용프로그램은 데이터를 고유형식에 맞게 관리

- 응용프로그램마다** 독립적**으로 관리

#### 파일시스템의 단점

- 데이터가 중복되서 저장됨

- 여러 데이터 중에서 변경된 하나의 데이터를 찾는데 오래걸린다

- 프로그램에 종속된 데이터를 다른 목적을 프로그램에서 사용하기 어려움

- 파일자체에서는 데이터 보안을 유지하기 어려움

## 3. 데이터베이스의 도입배경

### 01      데이터베이스

#### 도입배경

- 파일 시스템의 단점 극복

- 다수 사용자의 정보 공융

#### 개념

> 다수의 사용자가 데이터를 공유하여 사용할 수 있도록 통합하여 저장한 데이터 집합

| **장점** | **단점** |
| --- | --- |
| - 데이터 공유가 쉬움**- 중복된 데이터를 최소화- 데이터의 무결성**을 유지**- 데이터 보안을 유지****- 데이터를 표준화**시킬수있음**- 데이터 저장공간을 절약**할 수 있음 | - **전문가** 필요**- 데이터 백업과 복구 어려움**- 소프트웨어 구입,전문가 고용을 위한 비용 발생- 다수의 사용자가 동시사용할때 집중된 데이터 사용에   과부하가 발생할 수 있음 |

## 4. 데이터베이스 관리 시스템

### 01      데이터베이스 관리 시스템

**DBMS** : **D**ata**B**ase **M**anagement **S**ystem

**필요한 데이터를 데이터베이스에 정의,생성,조작,공유 할수있도록 편한 기능을 제공하는 시스템 소프트웨어**

#### 특징

**  01  ** 데이터 **중복 및 불일치 최소화**

**  02  ** 데이터 **독립성 유지** : 데이터베이스 내의 데이터 변화가 응용프로그램에게 영향을 끼치지 않도록 한다

**  03  ** 데이터 **무결성 유지** 가능 : 무결성 위반 시 데이터는 저장이 거절됨

**  04  ** 데이터를 **표준화** 시킬 수 있음 : 데이터의 명칭, 정의 형식, 규칙을 적용하는 것(표준화)

**  05  ** 데이터 **보안 향상** : 접근권한을 사용자에게 부여하여 접근 제한 할 수 있음

**  06  ** 데이터 **공유가 쉬움**

**+ 부과적인 효과**

**응용프로그램개발 비용과 인건비가 감소됨**

**   단점   **

- 백업이 복잡함

- 운영하기위한 전문 인력 필요

#### 데이터관리 패러다임 변화

| 파일시스템의 데이터 관리 | DBMS의 데이터 관리 |
| --- | --- |
| - 데이터 중복 발생- 응용 프로그램이 데이터를 직접 관리- 데이터형식이 응용 프로그램마다 다름- 데이터가 특정 응용 프로그램에 종속됨 | - 하나의 DBMS가 관리하므로 데이터 중복을 피할수 있다- DBMS만 데이터 관리- DBMS가 통일된 데이터형식을 사용- 응용 프로그램과는 별개로 데이터가 관리 됨 |

## 5. 데이터베이스 모델

### 01      데이터베이스 모델

- 컴퓨터에 데이터를 저장하는 방식

- 연관데이터를 사용하기 쉽고 구조적으로 표현한 방식

**  01  ** **계층형** 데이터베이스 모델(조직도 형태, 지금은 잘 사용되지 않음)

**  02  ** **네트워크형** 데이터베이스 모델(망형,개체간의 관계를 그래프 구조로)

**  03  ** **관계형** 데이터베이스 모델(테이블형태로 구현 : 오라클,mysql, 엑셀 등)

- 중복이될 수 있는데이터는 별개의 테이블로 정의하여 연결

- 현재 가장 많이 사용하는 데이터베이스 모델

**  04  ** **객체형** 데이터베이스 모델

- 객체지향 프로그래밍에서 사용되는 객체 개념을 기반으로함

- 실질적으로 db에 적용이 쉽지않아 상용 dbms는 많지않다
